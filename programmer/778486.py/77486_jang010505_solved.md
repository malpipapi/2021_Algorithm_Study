## 코드 풀이
배열과 map(dict)를 이용하면 간단하게 풀리는 문제이다.

### 배열을 사용하는 이유
다단계의 구조를 보면 트리의 형태를 뛴다. 
또한 자식노드와 부모노드의 상호작용을 보면 자식노드 -> 부모노드의 형태를 뛰게 된다. 다르게 말하면 자식노드의 부모노드만 알아도 된다는 것이다.
<details>
<summary>자식노드와 부모노드의 관계</summary>
<div markdown="1">
판매책의 수익은 부모노드인 보스에게만 가기 때문이다. 또한 판매책의 수익은 자식노드인 추천인에게 안 가기 때문이다.
</div>
</details>

#### test case 1에서의 예시
center를 0번째 노드라 하고 enroll의 서순으로 번호를 넣는다.<br/>
parents = [-1, 0, 0, 2, 3, 2, 3, 5, 3]<br/>
center의 부모노드는 없기에 -1 <br/>
jokn, mary의 부모노드는 center인 0<br/>
edward, emily, jaimie의 부모노드는 mary인 2<br/>
sam, young의 부모노드는 edward인 3<br/>
tod의 부모노드는 jaimie인 5<br/>

### map(dict)를 쓰는 이유
enroll, referral, seller에서는 정수가 아닌 문자열이 들어오기 떄문에 매번 for문으로 매칭하기엔 시간적으로 낭비이다. 그러므로 사전형을 사용해서 각 이름 별 인덱스를 지정해서 사용한다.
